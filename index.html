How It Works

Five steps. One pipeline.
Zero blind spots.

From raw supplier data to automated mitigation — LogiTwin handles every step of the supply chain intelligence loop.

1
Ingest
Suppliers, factories, ports ingested via REST API into Neo4j with Pydantic v2 validation
2
Detect
Disruption event fired — port congestion, supplier offline, capacity degradation
3
Traverse
BFS graph traversal finds every downstream factory via SUPPLIES and SHIPS_THROUGH paths
4
Score
Impact formula: severity × transit_days × (1−capacity_rate). CRITICAL/HIGH/MEDIUM/LOW
5
Mitigate
Hook registry fires async carrier API calls — reroute, activate backup, notify procurement
Graph-Native Architecture
Supply chains are graphs — not tables. Neo4j stores every supplier, factory, and port as nodes with typed relationships. Traversal queries that would take seconds in SQL run in milliseconds on the graph.
