How It Works

From disruption to mitigation
in one pipeline

LogiTwin models your supply chain as a live graph. When any node fails, the engine traverses every downstream path and scores the impact — automatically.

📡 1
Ingest
Suppliers, factories, ports and all relationships ingested via REST API into Neo4j graph database with strict Pydantic v2 validation.
⚡ 2
Detect
Disruption event fired — port congestion, supplier offline, capacity degradation, or weather strike. Celery worker picks it up instantly from Redis.
🕸️ 3
Traverse
BFS graph traversal finds every factory downstream of the disrupted node — via direct SUPPLIES paths and maritime SHIPS_THROUGH routes simultaneously.
📊 4
Score
Impact formula: severity × transit_days × (1 − capacity_rate). Every factory ranked CRITICAL / HIGH / MEDIUM / LOW. Results written to Neo4j.
🛡️ 5
Mitigate
Hook registry fires async carrier API calls — reroute via alternate port, activate backup supplier, notify procurement. Audit trail written to graph.

Graph Database

Your supply chain.
Modeled as a graph.

Neo4j stores every supplier, factory, and port as a node. Every delivery route as a relationship. When Hamburg port goes down, the engine traverses every path downstream in milliseconds — finding BMW, Mahindra, Tata before your procurement team even knows.

  • (:Supplier) — 31 nodes, Tier 1/2/3
  • (:Factory) — 11 assembly plants
  • (:Port) — 8 global ports
  • [:SHIPS_THROUGH] — maritime paths
  • [:IMPACTS] — disruption → factory
