Scoring Engine

The impact formula.

Every factory in the graph is scored using a deterministic formula that captures severity, supply chain length, and factory vulnerability simultaneously.

Impact Score Formula
impact_score = severity_input × max(transit_days, 1) × max(1.0 − capacity_rate, 0.05)
severity_input
0.0 to 1.0 — how badly the disrupted node is affected. 1.0 = complete failure (TSMC offline). 0.75 = partial (Busan 75% congested).
transit_days
Total days in the disrupted supply path. Hamburg to Chennai = 24 days. That's why BMW scores highest — longest path, most days at risk.
(1 − capacity_rate)
Factory vulnerability. BMW at 70% capacity scores 0.30 vulnerability. Maruti at 92% scores 0.08. Lower capacity = higher exposure to disruption.
Risk Thresholds
Risk Level Impact Score Range Real Example Recommended Action
● CRITICAL ≥ 5.0 BMW India — 6.75 (Hamburg blocked) Immediate automated mitigation
● HIGH ≥ 2.0 Honda India — 4.60 (Renesas offline) Urgent — trigger SS3 mitigation
● MEDIUM ≥ 0.5 Maruti Suzuki — 1.94 (multi-path) Monitor — queue mitigation options
● LOW < 0.5 Direct factory hit — no cascade Log and watch
API Reference

23 endpoints.
Full REST surface.

Every operation in LogiTwin is accessible via a documented REST API. JWT Bearer token authentication on all endpoints.

POST
/ingest/supplier
Add a supplier node with tier, location, and ID. MERGE-based — safe to re-ingest.
GET
/ingest/suppliers
List all supplier nodes currently in the Neo4j graph.
POST
/ingest/factory
Add an assembly factory with capacity_rate and location.
GET
/ingest/factories
List all factory nodes with current capacity rates.
POST
/ingest/port
Add a port node with clearance days and operational status.
GET
/ingest/ports
List all port nodes with status and clearance times.
POST
/ingest/relationship/supplies
Create a SUPPLIES relationship between supplier and factory with lead_time_days.
POST
/ingest/relationship/ships-through
Create a maritime path: supplier → port → factory with transit days on each leg.
POST
/simulation/run
Fire a custom disruption simulation. Returns task_id immediately. Non-blocking.
GET
/simulation/status/{task_id}
Poll simulation result. Returns full impact report when status = SUCCESS.
POST
/simulation/inject/{1,2,3}
Inject predefined mock scenario. Instant demo trigger.
GET
/simulation/events
List all DISRUPTION_EVENT nodes ordered by timestamp descending.
GET

Get ranked reroute candidates for a disruption event. Combines hook registry + graph paths.

Graph Schema

The graph that powers everything.

Five node types. Five relationship types. Every supply chain event permanently recorded as a graph traversal.

(:Supplier)
(:Factory)
(:Port)
(:DISRUPTION_EVENT)
[:SUPPLIES] → component_id, lead_time_days · Supplier to Factory direct delivery
[:SHIPS_THROUGH] → transit_days · Supplier to Port leg of maritime route
[:DELIVERS_TO] → transit_days · Port to Factory delivery leg
[:IMPACTS] → impact_score, risk_level, path_type · Written by simulation engine on every run
[:MITIGATION_ACTION] → action_id, carrier_name, success, duration_ms, executed_at · Full audit trail
